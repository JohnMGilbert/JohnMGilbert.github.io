Asset and Animation Manager

Description & Motivation

My Asset and Animation Manager is a custom tool built in the Godot engine designed to simplify the process of importing, organizing, and animating game assets. Its primary goal is to give artists and non-programmers more control over how their assets appear and behave in-game, without needing to navigate Godot’s complex interface or rely on code. By bridging the gap between artist and programmer, the application promotes a smoother, more collaborative workflow.

Many artists struggle to translate their creative intentions into functioning game assets because existing animation tools either require programming knowledge or focus too heavily on image editing rather than animation management. This project addresses those issues by offering an intuitive workspace where users can drag and drop assets, organize them visually, and export complete animation data packages directly into Godot projects.

Key Features

  • Asset Importing: Drag-and-drop and directory-based importing for images and sounds.
  • Organization Tools: Tagging, metadata editing, and search features to keep assets easily accessible.
  • Animation Timeline: Visual drag-and-drop frame sequencing with a real-time preview window.
  • Audio Integration: Sync sound effects to animation frames with live playback testing.
  • Metadata Export: Generate JSON (JavaScript Object Notation) packages for seamless integration into Godot projects.
  • User-Focused Design: Intuitive interface, minimal clutter, and responsive controls suited for hobbyists and small creative teams.

Technical Overview

The application is developed entirely within the Godot engine, ensuring compatibility with existing Godot projects. It supports PNG (Portable Network Graphics), MP4 (MPEG-4 Part 14), and WAV (Waveform Audio File Format) formats and is designed to remain lightweight and responsive even when managing hundreds of assets. By abstracting complex animation logic into a visual editor, it allows artists to focus on creativity while maintaining technical accuracy.

Intended Use

The Asset and Animation Manager is meant for small, independent development teams and individual artists working on 2D games. It can be used during the asset creation phase to prepare fully configured animations before importing them into a game project, allowing programmers and artists to collaborate efficiently without technical bottlenecks.

Online Study & Prototyping Phase

The project followed a user-centered design process beginning with an early prototype and online-style feedback sessions. An initial low-fidelity interface was created to explore the basic workflow: creating a project, importing sprites, building an animation, and exporting it.

Participants completed three core tasks while I observed their actions and collected questionnaire responses:

  • Create a new project and bring an asset into the scene.
  • Link the project to a Godot repository.
  • Export a packaged animation sequence.

These early sessions highlighted confusion around where to drag sprites, how export and repository linking worked, and which stage of the process the user was currently in. This feedback directly informed the design of the higher-fidelity prototypes and the layout of the main interface.

User Study 1 Results

User Study 1 evaluated the early prototype with six participants ranging from novice users to experienced animators. Each participant completed the three core tasks while I recorded task time, clarifying questions, and interaction patterns. Afterward, they completed a questionnaire using 10 Likert-scale items (1 = strongly disagree, 5 = strongly agree) focused on layout clarity, import and export understanding, workflow logic, and overall approachability.

On average, participants rated the tool around 3.7 out of 5 overall. They generally found the layout and number of controls manageable, but reported confusion about:

  • Where to drop sprites when building an animation.
  • What the export step would produce and where it would go in Godot.
  • Which stage of the process they were currently in.

Open-ended responses requested clearer visual cues (such as drop targets and signals), simple tutorials or tooltips, and more feedback about what actions were possible at each step.

User Study 2 Results

User Study 2 was conducted on a functional Godot-based version of the Asset and Animation Manager after incorporating findings from User Study 1. The same three tasks and questionnaire were reused to make comparison possible, and again six participants with varied experience levels were observed.

The overall average rating stayed around 3.7 out of 5, but the pattern of responses shifted. Participants reported that the workflow felt more coherent and that it was easier to move from importing sprites to exporting a complete animation. Task completion became more consistent: all participants successfully completed all three tasks, and export actions in particular became very fast and direct.

Remaining issues appeared mostly at the “polish” level. Novice users sometimes still needed stronger guidance about what the preview window represented or where to start when dragging sprites. Participants suggested clearer section labels (for example “Frame Builder” and “Preview”), highlighting the currently selected frame, zoom controls, and small onboarding aids such as an introductory popup or a help icon.

Iterative Design & Takeaways

Comparing User Studies 1 and 2 shows how the design evolved in response to real user data. Early pain points around drag-and-drop, export behavior, and workflow visibility led to changes in layout, stronger visual cues, clearer tab naming, and a more structured build-versus-export separation.

By the second study, participants spent less time asking “How do I use this?” and more time suggesting enhancements such as keyboard shortcuts, better frame control, and advanced navigation features. This shift indicates that the core interaction model has become understandable, leaving room for the tool to grow into a more powerful production-ready system.
